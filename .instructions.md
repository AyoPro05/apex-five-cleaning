# Apex Five Cleaning - Developer Instructions

## Project Context

**Apex Five Cleaning** is a professional cleaning services web application built with:
- **Frontend**: React 18 + Vite + Tailwind CSS (SPA)
- **Backend**: Node.js + Express + MongoDB
- **Payments**: Stripe API integration
- **Auth**: JWT (customers), Bearer tokens (admin)
- **Testing**: Vitest (client), Jest-compatible (server)

## Architecture Overview

```
apex-five-cleaning/
├── client/          # React SPA (Vite)
├── server/          # Express API
├── docs/            # Reference docs (deployment, security, SEO, email)
└── README.md        # Project overview
```

## Development Stack

| Layer | Technology | Key Details |
|-------|-----------|------------|
| **Frontend** | React 18, Vite, Tailwind CSS | Hot module reloading, tree-shaking |
| **Backend** | Express.js, Node 18+ | RESTful API, middlewares, route handlers |
| **Database** | MongoDB (Atlas) | Mongoose ODM for schema modeling |
| **Auth** | JWT + Bearer tokens | `authMiddleware` for protected routes |
| **Payments** | Stripe SDK | Quote payments, subscription handling |
| **Email** | SendGrid + Nodemailer | Transactional emails, notifications |
| **Testing** | Vitest (client), built-in (server) | Unit tests with React Testing Library |

## Running Locally

### Backend
```bash
cd server
npm install
cp .env.example .env          # Configure MongoDB_URI, Stripe keys, SendGrid, etc.
npm run dev                    # Starts on :5001
```

### Frontend
```bash
cd client
npm install
cp .env.example .env          # Configure VITE_API_URL=http://localhost:5001
npm run dev                    # Starts on :5173
```

## Code Conventions

### React Components
- Functional components with hooks
- Props validation via TypeScript comments or JSDoc
- Component files: `PascalCase.jsx`
- Styles: Tailwind classes inline

### Node/Express
- Route handlers in `routes/`
- Business logic in `controllers/`
- Database models in `models/`
- Middleware in `middleware/`
- Arrow functions for consistency

### Naming
- **Files**: `camelCase.js` (utils), `PascalCase.jsx` (components)
- **Variables/Functions**: `camelCase`
- **Classes/Components**: `PascalCase`
- **Constants**: `UPPER_SNAKE_CASE`

## API Conventions

All endpoints are RESTful:
- `POST /api/auth/signup` - Register customer
- `POST /api/auth/login` - Login
- `GET /api/quotes` - Get quotes (admin only)
- `POST /api/bookings` - Create booking
- `POST /api/payments` - Process payment
- `GET /api/admin/...` - Admin endpoints (Bearer token required)

## Security

- JWT stored in HTTP-only cookies (recommended)
- Bearer tokens for admin endpoints
- Rate limiting on auth endpoints
- Input validation with Joi
- CORS configured for cross-origin requests
- See `docs/SECURITY.md` for full details

## Common Tasks

### Adding a New API Endpoint
1. Create route handler in `routes/`
2. Add controller logic in `controllers/`
3. Use appropriate middleware (auth, validation)
4. Test with REST Client

### Adding a New React Component
1. Create in `components/` or `pages/`
2. Import dependencies (React, icons from lucide-react)
3. Use Tailwind for styling
4. Export and import in App.jsx or route

### Modifying the Database Schema
1. Update Mongoose model in `models/`
2. Run migration if needed (see `scripts/`)
3. Update API handlers
